## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** (v18 or higher recommended)
- **pnpm** package manager
- **Git** for version control

<Note>
  While npm and yarn will work, this project uses pnpm for dependency management. Install it globally with `npm install -g pnpm` if needed.
</Note>

## Installation

Follow these steps to set up Velaria on your local machine:

<Steps>
  <Step title="Clone the repository">
    Clone the Velaria source code to your local machine:
    
    ```bash
    git clone <repository-url>
    cd velas-aromaticas
    ```
    
    <Tip>
      The project directory is named `velas-aromaticas` as specified in package.json.
    </Tip>
  </Step>

  <Step title="Install dependencies">
    Install all required packages using pnpm:
    
    ```bash
    pnpm install
    ```
    
    This will install:
    - Astro 5.16.6 and Vercel adapter
    - Tailwind CSS 4.1.13 with animation plugins
    - SweetAlert2 for notifications
    - All other dependencies listed in package.json
    
    <Note>
      The installation creates a `pnpm-lock.yaml` file to ensure consistent dependency versions across environments.
    </Note>
  </Step>

  <Step title="Start the development server">
    Launch the Astro development server:
    
    ```bash
    pnpm dev
    ```
    
    The development server will start at `http://localhost:4321`.
    
    <Tip>
      The dev server includes hot module replacement (HMR) - changes to components will reflect immediately without a full page reload.
    </Tip>
  </Step>

  <Step title="Open in your browser">
    Navigate to `http://localhost:4321` to see Velaria running locally.
    
    You should see the homepage with:
    - Hero header with parallax effect
    - Product catalog with animated reveals
    - Fragrances section
    - Contact form
    - WhatsApp integration
  </Step>
</Steps>

## Available commands

All commands are run from the root of the project:

| Command | Action | Description |
| :--- | :--- | :--- |
| `pnpm install` | Installs dependencies | Required before first run |
| `pnpm dev` | Starts local dev server | Runs at `localhost:4321` with HMR |
| `pnpm build` | Build production site | Outputs to `./dist/` directory |
| `pnpm preview` | Preview build locally | Test production build before deploying |
| `pnpm astro` | Run Astro CLI | Use for commands like `astro add`, `astro check` |

<Warning>
  The `pnpm build` command requires the Vercel adapter since the project is configured with `output: 'server'` for SSR.
</Warning>

```typescript Form submission handler
const form = document.getElementById("contact-form");
const btn = document.getElementById('btn-submit');

form.addEventListener("submit", async (e) => {
  e.preventDefault();
  
  btn.setAttribute('disabled', 'true');
  btn.innerHTML = 'Enviando...';
  
  const formData = new FormData(form);
  const data = Object.fromEntries(formData);
  
  const res = await fetch("/api/contact", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(data),
  });
  
  if (res.ok) {
    Swal.fire({
      title: "Gracias por tu mensaje",
      text: "Nos pondrémos en contacto contigo",
      icon: "success"
    });
    form.reset();
  }
});
```
</CodeGroup>

## Development workflow

Here's a typical development workflow:

<Steps>
  <Step title="Make changes">
    Edit component files in `src/components/` or pages in `src/pages/`.
    
    The dev server will automatically reload your changes thanks to HMR.
  </Step>

  <Step title="Test locally">
    Verify your changes in the browser at `localhost:4321`.
    
    Check:
    - Animations work correctly
    - Form submissions (requires API configuration)
    - Responsive design on different screen sizes
    - WhatsApp integration
  </Step>

  <Step title="Build for production">
    Test the production build locally:
    
    ```bash
    pnpm build
    pnpm preview
    ```
    
    This ensures SSR works correctly and catches any build-time errors.
  </Step>

  <Step title="Deploy to Vercel">
    Push your changes to trigger automatic deployment:
    
    ```bash
    git add .
    git commit -m "Your changes"
    git push
    ```
    
    Vercel will automatically build and deploy your changes.
  </Step>
</Steps>

<Tip>
  Use `pnpm astro check` to run TypeScript type checking and catch potential issues before deployment.
</Tip>
